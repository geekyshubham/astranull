### Safe-probe upgrade status (metadata_marker → bounded live probe)

Bounded, single-connection, metadata-only live probes now back these safe checks (impl in `src/lib/safeNetworkProbes.mjs`, simulation in `src/services/probeStub.mjs`, dispatch in `workers/probe-worker.mjs`; all use injectable transports so unit tests never touch the network):

| Check | Probe kind | Notes |
|---|---|---|
| `dns.secondary_failover.safe` | `dns_resolve` | Bounded NS/SOA resolution via the DNS vector-family worker path. |
| `tls.idle_connection_timeout.safe` | `tls_session` | One bounded TLS handshake (node:tls); records negotiated protocol/cipher/authorized, then closes. No application data. |
| `protocol.http2_stream_concurrency.safe` | `http2_settings` | One bounded h2 session (node:http2); reads advertised SETTINGS (max concurrent streams), then closes. No request stream, no flood. |

Deliberately kept `metadata_marker` (safety / feasibility judgment per `AGENTS.md` safe-by-default and "do not ship attack tooling casually"):

| Check | Why not a live probe |
|---|---|
| `dns.zone_transfer_exposure.safe` | A live AXFR attempt transfers zone data when the server is misconfigured; no safe node builtin for AXFR framing. Exposure intent stays metadata + SOC-gated confirmation. |
| `l7.login_abuse_flow.safe` | Live login attempts risk account lockout / disruption on the customer's real auth flow — not safe as a self-serve probe. |
| `l7.graphql_complexity.safe` | The complex/nested query *is* the abuse vector; a live probe would itself be the disruptive load. |
| `protocol.grpc_reflection_stream.safe` | Requires protobuf + h2 stream framing (heavy new dependency); benign value low relative to added surface. |
| `protocol.websocket_connection_controls.safe` | Requires a WebSocket handshake/library; deferred to keep the safe tier dependency-free. |
