AstraNull Check Library

Reference for every check in CHECK_CATALOG (65 entries). Safe checks are customer-runnable with bounded probes. SOC-gated checks require authorization and SOC execution. Generated from the repo catalog — re-run node scripts/generate-check-library-html.mjs after catalog changes.

How a check runs

  1. Declare target group + target (FQDN/IP/URL) and optional expected behavior.
  2. Start test run — planner picks check, creates signed probe job(s).
  3. Probe worker executes bounded probe (probe_profile) against declared target only.
  4. Agent (if deployed & required) uploads metadata observation when local signal matches.
  5. Correlation compares external_result + observation vs expected_behavior → verdict + finding.

Glossary

TermMeaning
Probe workerAstraNull-signed outside process that runs bounded network probes against customer-declared targets (not in your app process).
AgentCustomer-deployed outbound observer (heartbeat, packet, mirror, log-tail, or canary). Correlates inside traffic with outside probes — not required for every check.
expected_behaviorWhat you declare should happen when probed (e.g. block before origin). Verdict compares probe result to this.
probe_profile.kindThe bounded probe technique used (HTTP HEAD, TCP connect, DNS lookup, etc.).
external_resultProbe outcome: blocked, timeout, connected, or allowed.
agent_observationMetadata that the agent saw matching probe traffic at the observation point.
metadata_markerCatalog-only simulation in dev/CI; production signed-worker may still dispatch a minimal stub unless a live probe kind is set.
soc_gatedNot runnable from customer UI — requires SOC approval, authorization pack, and governed execution.
nonce_hashCorrelation token linking probe job to agent observation without sending raw payloads.
direct_origin_ipCustomer-declared origin IP used for Host/SNI bypass probes (no automatic origin hunting).

Expected behavior values

must_block_before_originAttack-ish probe should be stopped at edge/WAF — must not reach your app.
must_reach_canaryTraffic should reach your opt-in canary on the protected path (positive test).
must_allow_baseline_healthLegitimate health probe should succeed (declared; correlation partial).
must_challenge_or_rate_limitSuspicious probe should get challenge or 429 (declared; correlation partial).
must_not_expose_direct_ipOrigin IP should not answer when only CDN hostname is public (declared; correlation partial).
