# Reports Page

## Report types

| Report | Audience | Contents |
|---|---|---|
| Executive Readiness Report | CISO/board | Score, trend, critical gaps, business-service risk, actions. |
| Technical Evidence Report | Engineers/security | Checks, target groups, probes, observations, verdict logic, remediation. |
| SOC High-Scale Report | SOC/customer/security | Approval trail, run timeline, metrics, health, stop events, outcome. |
| Audit Evidence Pack | Auditors/compliance | Immutable test IDs, timestamps, authorization docs, evidence, retention policy. |
| Agent Placement Report | Platform teams | Agents, placements, visibility gaps, recommended moves. |

## Report builder fields

| Field | Description |
|---|---|
| Time range | Last 7/30/90 days or custom. |
| Target groups | All or selected. |
| Severity filter | Critical/high/medium/low. |
| Include raw evidence | Yes/no. |
| Include accepted risk | Yes/no. |
| Include SOC approvals | Yes/no. |
| Format | HTML, PDF, Markdown, JSON. |

## Executive report structure

1. Summary score.
2. Business service readiness table.
3. Critical gaps.
4. What changed since last report.
5. High-scale readiness status.
6. Remediation priorities.
7. Evidence appendix.

## Technical report structure

1. Scope and target groups.
2. Agent placement map.
3. Check catalog and versions.
4. Run results.
5. Evidence details.
6. Findings and remediation.
7. Inconclusive items.
8. Retest plan.

## Completion criteria

Reports are complete when users can generate, preview, export, share, and audit evidence packs from the UI.
