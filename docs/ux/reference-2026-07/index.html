A defensive readiness platform, not self-service attack tooling.

Three commitments shape every screen, every probe, every verdict in AstraNull.

No-access-first

You declare the target groups you want validated. AstraNull never defaults to cloud access and never auto-discovers your IP inventory. Validation uses outside probes and inside agents you place. Nothing more.

Evidence over assumptions

Every verdict is backed by correlated probe results and agent observations, written to an evidence vault you control. Readiness is a number you can defend in an incident review, not a green checkmark.

SOC-gated high-scale

Default validation is low-volume, bounded, and non-disruptive. High-scale assessments are reviewed and executed by the AstraNull SOC after approval. Customers submit requests, never run floods themselves.

The loop

Declare. Validate. Evidence. Govern.

A four-stage loop that turns a declared scope into a defensible readiness posture.

  1. 01

    Scope your target groups

    Register environments and target groups: the FQDNs, DNS zones, and TCP surfaces you want validated. Declare expected behavior so verdicts map to your real edge topology.

  2. 02

    Place agents, run safe checks

    Install outbound-only agents and run the safe-by-default check catalog: origin-bypass, L3/L4, DNS, L7/API. Each check is bounded and metadata-only unless you escalate.

  3. 03

    Correlate probe + agent

    Verdicts combine external probe reachability with internal agent path observation. Every result lands in the evidence vault, exportable for audits and incident reviews.

  4. 04

    Escalate through the SOC

    When you need high-scale validation, submit a request. The SOC reviews, schedules, and executes under a kill switch, with full custody and audit trail.

Fastest path. Installs the systemd service on a host you already run. Requires only outbound HTTPS to the AstraNull control plane.
# Outbound-only. No inbound ports opened. curl -fsSL https://get.astranull.dev/agent/install.sh \ | sudo ASTRANULL_API_URL="https://astranull.example" \ ASTRANULL_BOOTSTRAP_TOKEN="<bootstrap-token>" \ ASTRANULL_TARGET_GROUP="tg_checkout" bash

Bootstrap token is redeemed for a long-lived agent credential and deleted. The credential lives in /var/lib/astranull/identity.json at mode 0600.

Distroless Node 22 image. Non-root (uid 10001). No secrets baked into layers. Same binary as the host agent; behavior is identical.
