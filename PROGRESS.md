# AstraNull Progress Tracker

Use this file as the single build tracker. Each task links to the doc that explains how it must be implemented.

Status: `[ ]` not started, `[~]` in progress, `[x]` complete, `[!]` blocked, `[?]` needs decision.

## Production readiness rule

**Do not mark `[x]`** unless the linked implementation doc acceptance criteria are met **and** the applicable gates in [`docs/release-checklist.md`](docs/release-checklist.md) have verification evidence (tests, staging run, security review). Developer validation behavior (local JSON store, header auth, probe simulation, adapter dry-run) does **not** satisfy production completion goals.

When a task is partially implemented for local engineering only, keep `[~]` and state the **production release blocker** in the completion goal.

Metadata-only evidence validators (`npm run â€¦:evidence`, `npm run release:gap-audit`) check JSON contracts and forbidden-field rules only. They do **not** substitute for staging execution, security review, SOC/legal signoff, KMS/vault custody, or provider evidence. `npm run release:sample-evidence` writes rehearsal fixtures (`rehearsal_only: true`) for operator walkthroughs â€” never treat passing sample generation or local validators as production readiness.
